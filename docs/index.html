workspace-cli

把一个需求涉及的多个 Git 仓库集中到同一个 workspace 中，用统一 feature 分支完成跨仓库开发，再把多个 ready 需求集成到 release 分支并发布到 base branch。

查看源码 一键安装 
$ workspace req create "支付链路优化" \
  --key pay-flow \
  --repo backend \
  --repo frontend

pay-flow  ~/.workspace-cli/work/requirements/pay-flow

$ workspace dev pay-flow --tool codex
$ workspace ide pay-flow
$ workspace req finish pay-flow -m "feat: complete pay-flow"
$ workspace release create "2026-07-01 发布" --key 2026-07-01 --req pay-flow
$ workspace release integrate 2026-07-01
$ workspace release publish 2026-07-01 --tested

为跨仓库需求开发而生

workspace-cli 的 workspace 不是单个代码仓库，而是围绕一个需求临时组织起来的开发空间。它负责把多个 repo 拉进同一个目录，保持统一 feature 分支，在需求完成时进入 ready 状态，并把多个 ready 需求集成到 release 分支后发布到 base branch。

install

复制一行命令安装最新版本

安装脚本会识别 macOS/Linux 和 CPU 架构，从 GitHub latest release 下载对应包，校验 checksum 后安装 `workspace`。 

curl -fsSL https://idefav.github.io/workspace-cli/install.sh | sh

默认安装到 /usr/local/bin。用户态安装可使用 INSTALL_DIR="$HOME/.local/bin"。

completion

启用 Shell 自动补全

workspace-cli 可以为 bash、zsh、fish 和 PowerShell 生成补全脚本。安装后执行对应命令，把脚本写入 shell 的 completion 目录。

zsh

mkdir -p "$HOME/.zsh/completions"
workspace completion zsh \
  > "$HOME/.zsh/completions/_workspace"
exec zsh

fish

mkdir -p "$HOME/.config/fish/completions"
workspace completion fish \
  > "$HOME/.config/fish/completions/workspace.fish"

workflow

从需求开发到 Release 发布

01

注册仓库

把需要管理的 Git repo clone 为本地 bare repo，记录 remote、base branch 和 URL。

02

创建需求

选择多个 repo，创建统一 feature 分支，并把 worktree 集中到同一个需求目录。

03

进入开发

使用 Codex、Claude Code 或 IDE 从需求 workspace 启动，围绕同一个需求修改多个仓库。

04

完成为 ready

全部 repo feature 分支 push 成功后再删除需求 worktree，失败时保留现场并允许 finish 重试。

05

集成测试

选择多个 ready 需求，删除并重建 release 分支，按顺序合入 feature 分支后测试。

06

发布归档

测试确认后把 release 合并到每个 repo 的 base branch，发布成功的需求进入 completed 并归档。

commands

v1 命令面

Repo 管理

  • workspace repo add <name> <url>
  • workspace repo list --all
  • workspace repo sync [name]
  • workspace repo update <name>
  • workspace repo remove <name>

需求管理

  • workspace req create <title> --repo <name>
  • workspace req list [--all]
  • workspace req show <key>
  • workspace req update <key> --title <title>
  • workspace req add-repo <key> <repo>
  • workspace req archive <key>
  • workspace req finish <key> -m <message>
  • workspace req reopen <key>
  • workspace dev <key> --tool codex|claude
  • workspace ide <key> --tool vscode

Release 管理

  • workspace release create <name> --req <key>
  • workspace release list [--all]
  • workspace release show <key>
  • workspace release status <key>
  • workspace release add-req <release> <req>
  • workspace release remove-req <release> <req>
  • workspace release integrate <key>
  • workspace release publish <key> --tested

工具

  • workspace update --check
  • workspace completion zsh

design

保守、可恢复、只管开发流程

SQLite 状态库

记录 repo、需求、绑定快照、worktree path、状态和失败 operation log。

Git worktree 编排

托管 repo 使用 bare clone，需求 workspace 使用 worktree，避免重复 clone。

两阶段完成体验

需求 finish 后进入 ready；release publish 成功后才写入 completed 并归档。
